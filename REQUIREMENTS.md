# Lexio Underground — Requirements

---

## V1 Requirements (4–6 Week Target)

### AUTH-001 — Authentication
- Magic link (email) login via Supabase
- Google OAuth via Supabase
- Role system: `student` | `teacher` | `admin`
- Session persistence, refresh token handling
- Post-auth redirect: onboarding flow (new) or dashboard (returning)

---

### PAY-001 — Subscription
- Stripe monthly subscription product
- Free tier: placement test only (no exercises, no tutor)
- Paid tier: full platform access
- Stripe webhook → Supabase `subscriptions` table sync
- Customer portal link (manage/cancel) via Stripe Billing Portal
- Subscription gate: middleware redirects free users to upgrade page

---

### PLACE-001 — CEFR Placement Test
- 15-question adaptive test, generated by Claude on first login
- Questions span A1–C2 across grammar and vocabulary
- Adaptive logic: correct answer → harder next question; wrong → easier
- Result assigns student to one of: A1, A2, B1, B2, C1, C2
- Level stored in `user_profiles.cefr_level`
- Retake available after 30 days (teacher can force reset)
- Placement test available to free users (lead capture mechanism)

---

### EXERCISE-001 — Exercise Engine

#### Exercise Types (v1)
| Type | Description |
|------|-------------|
| `mcq` | Multiple choice — 1 correct of 4 options |
| `fill_blank` | Sentence with one or more gaps to fill |
| `vocab_flashcard` | Term shown → student selects correct definition |
| `vocab_drag` | Drag words into correct slots in a sentence |
| `listening_mcq` | TTS audio played → MCQ comprehension question |

#### Exercise Data Model
```
exercises {
  id uuid
  language text           -- 'en' | 'zh' (v2)
  cefr_level text         -- 'A1'|'A2'|'B1'|'B2'|'C1'|'C2'
  skill text              -- 'grammar'|'vocabulary'|'listening'|'reading'
  type text               -- see types above
  content jsonb           -- type-specific payload
  audio_url text          -- nullable, for listening_mcq
  status text             -- 'draft'|'approved'|'archived'
  ai_generated boolean
  generated_by text       -- model version
  created_at timestamptz
  reviewed_at timestamptz
  reviewed_by uuid
}
```

#### Answer Evaluation
- All v1 types are auto-evaluated server-side
- Results stored in `completions` table
- No open-ended graded writing in v1

### CONTENT-001 — AI Exercise Generation Pipeline
- Admin UI: input topic + CEFR level + exercise type → submit
- Claude API call with structured system prompt → returns JSON matching exercise schema
- Returned exercise lands in `status = 'draft'` review queue
- Admin review queue: preview rendered exercise → Approve / Reject / Edit inline
- Bulk generation: generate N exercises from a topic in one call
- TTS audio auto-generated for `listening_mcq` type (OpenAI TTS) on approval
- Generation logs stored for audit

---

### GAMIFY-001 — XP System
- XP awarded per exercise completion
- Base XP by level: A1=10, A2=15, B1=20, B2=25, C1=30, C2=40
- Difficulty multiplier: correct on first try = ×1.5
- XP stored in append-only `xp_log` table (never mutated, always summed)
- Level-up event fires when XP crosses CEFR thresholds
- XP visible on student dashboard (current total + progress to next threshold)

---

### GAMIFY-002 — Daily Streaks
- Streak increments on first exercise completion each calendar day (BRT timezone)
- Streak resets at midnight BRT if no activity
- Streak freeze: student gets 1 free freeze per 30 days (v1 simple implementation)
- Streak milestone badges at: 3, 7, 14, 30, 60, 100 days
- Streak at-risk email: Resend sends reminder at 20:00 BRT if no activity that day

---

### GAMIFY-003 — Leaderboard
- Weekly scope (resets Monday 00:00 BRT)
- Shows top 10 students by XP earned that week
- Student sees own rank even if outside top 10
- Powered by Supabase Realtime subscription
- Scoped to subscribed users only (no ghost free accounts inflating board)

---

### GAMIFY-004 — Badges & Achievements
| Badge | Trigger |
|-------|---------|
| First Step | Complete first exercise |
| On a Roll | 3-day streak |
| Week Warrior | 7-day streak |
| Fortnight | 14-day streak |

- Badge shelf visible on student profile

---

### TUTOR-001 — AI Text Chat Tutor
- Claude-powered conversational ESL tutor
- System prompt: ESL expert persona, adapts to student's CEFR level, never breaks character
- Student's level injected into system prompt dynamically
- Conversation history persisted in `chat_sessions` + `chat_messages` tables
- Max context window management: summarize older messages if approaching limit
- Tutor accessible from student dashboard (dedicated page)
- Free users: 3 tutor messages total (upgrade gate after)
- Rate limit: 20 messages per day for paid users (anti-abuse)

---

### TEACHER-001 — Teacher Dashboard
- View all assigned students (roster)
- Per-student view: CEFR level, XP, streak, completion rate, last active
- Assign exercise modules to individual students or cohort groups
- View assigned exercise completion status per student
- Manual CEFR level override (force placement reset)
- Export: student progress CSV (name, level, XP, completion %, last active)

---

### STUDENT-001 — Student Dashboard
- Level display (CEFR badge + label)
- XP progress bar (current → next level threshold)
- Streak counter with flame animation
- Today's queue: assigned exercises + recommended exercises
- Quick-access: AI Tutor, Leaderboard, Badge shelf
- Weekly XP chart (sparkline)

---

### ADMIN-001 — Admin CMS
- Exercise review queue (drafts pending approval)
- Exercise browser: filter by language, level, skill, type, status
- Student management: list, search, view profile, force actions
- Content generation trigger (topic + level + type → generate)
- Generation history log
- Platform stats: total users, active subscribers, exercises served today

---

### ONBOARD-001 — Onboarding Flow
1. Auth (magic link or Google)
2. Profile setup (name, professional context — optional)
3. Placement test (15 questions)
4. Level reveal screen (animated, shareable)
5. Subscription gate (if free, prompt upgrade; if paid, continue)
6. First exercise queue walkthrough (guided, 3 exercises)
7. Land on dashboard
