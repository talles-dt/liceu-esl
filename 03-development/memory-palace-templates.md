# Memory Palace Template Specification

## MVP Template Structure (Text-Based)

```
CONCEPT ‚Üí PALACE LOCATION ‚Üí VISUAL HOOK ‚Üí PT-BR LINGUISTIC ANCHOR
```

Every mnemonic generated by Leo must follow this exact four-part structure, separated by `‚Üí`. Validation rule: `mnemonic.split('‚Üí').length === 4`.

## Component Rules

| Component | Rules | Examples |
|-----------|-------|---------|
| **CONCEPT** | Target grammar or vocab item ‚Äî uppercase | `SUBJECT PRONOUNS`, `PRESENT PERFECT`, `PHRASAL VERBS` |
| **LOCATION** | Familiar PT-BR spatial anchor ‚Äî match to user interests | `KITCHEN`, `BUS COMMUTE`, `GYM`, `COFFEE SHOP` |
| **VISUAL HOOK** | Short action or image ‚Äî max 1 emoji | `steam rising from cup`, `clock with stuck hands`, `labeled shelves` |
| **PT-BR ANCHOR** | Portuguese word or phrase that sounds or conceptually relates | `'tenho' vs 'I am'`, `'fiz' vs 'I have done'`, `'desligar' vs 'turn off'` |

## Cognitive Load Guardrails

| Guardrail | Implementation | Why |
|-----------|---------------|-----|
| Max 1 mnemonic/lesson | Hard limit in prompt + post-generation validation | Prevents overwhelm |
| Reuse locations | Cycle through user's top 5 locations max | Builds familiarity over time |
| PT anchor mandatory | Validation: must contain Portuguese word or phrase | Leverages existing mental structures |
| No emoji overload | Max 1 emoji per mnemonic, from curated set of 20 | Visual consistency |
| Progressive complexity | Concrete nouns first, abstract grammar later | Matches CEFR progression |
